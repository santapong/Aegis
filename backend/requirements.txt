# Vercel-only Python deps for the experimentalServices backend.
#
# Vercel's Python builder picks this file up over pyproject.toml. The
# Docker / Render / local-dev path keeps using backend/pyproject.toml
# which includes the heavy deps (WeasyPrint + matplotlib + numpy +
# arq + asyncpg) that don't work on Vercel's serverless Python
# runtime.
#
# Endpoints that need the heavy deps return 503 cleanly when run on
# Vercel:
#   - /api/reports/export.pdf  → 503 "WeasyPrint not installed"
#   - /api/jobs/*              → 503 "Job queue not configured"
#     (and CACHE_BACKEND=memory keeps everything else functional)
#
# CSV export, dashboard, all CRUD, AI calls, Google sign-in, exports
# (NDJSON), Stripe, cache (in-memory), rate limiter (in-memory): all
# work on Vercel.
#
# Keep this file in sync with pyproject.toml's `dependencies` list,
# minus the heavy-dep lines.
